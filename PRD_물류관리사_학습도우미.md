# PRD: 물류관리사 학습 도우미
**AI 바이브 코딩 해커톤 · 팀 5인**

---

## 1. 문제 정의 (Problem Statement)

### 배경
물류관리사는 물류관리론·화물운송론·보관하역론·국제물류론·물류관련법규 5개 과목, 과목당 40문항 5지선다형 시험이며, 매 과목 40점 이상 + 전 과목 평균 60점 이상을 받아야 합격한다. 기본서 기준 학습 분량이 약 1,300페이지에 달하고, 시험은 연 1회뿐이라 재도전 기회가 제한적이다. 합격률도 해마다 26%~51%로 편차가 커서 어느 과목/영역에 집중해야 할지 판단이 어렵다.

### 우리가 발견한 문제
1. **방대한 암기량 vs 한정된 학습 시간** — 순서대로 다 읽기엔 시간이 부족하다
2. **법령 과목의 정확성 부담** — 조문이 복잡하고, 비슷한 개념(예: 보세구역 vs 자유무역지역)이 헷갈린다
3. **취약점 파악의 어려움** — 어느 과목/개념에서 반복적으로 틀리는지 스스로 추적하기 어렵다
4. **질문할 곳의 부재** — 독학 시 "이게 왜 정답이지?"를 즉시 물어볼 사람이 없다

### 해결 가설
> "전체를 순서대로 암기하는 방식"에서 "필요한 개념을 정확한 근거와 함께 즉시 찾아 이해하는 방식"으로 전환하면, 학습 효율과 신뢰도를 동시에 높일 수 있다.

## 2. 타겟 사용자

- 물류관리사를 준비하는 대학생 (전공 무관, 특히 독학하는 수험생)
- 페르소나: "전공 공부, 대외활동과 병행하며 자격증을 준비하는 3~4학년"

## 3. MVP 정의

### 3.1 스코프 원칙
- **5과목 전체가 아니라 1~2과목(우선 물류관련법규 + 물류관리론)에 집중**해 완성도를 높인다
- 나머지 과목은 "구조는 동일, 데이터만 채우면 확장 가능"함을 발표에서 강조

### 3.2 MVP 필수 기능 (1일차 목표)

| 기능 | 설명 | 우선순위 |
|---|---|---|
| **문서 DB 구축** | 법령 원문 + 개념 정리 문서를 코퍼스로 적재 | P0 |
| **개념/법령 Q&A** | 질문 시 문서에서 근거를 찾아 답변 (근거 원문 함께 표시) | P0 |
| **기출문제 풀이** | 과목별 문제 제공, 정답/오답 체크 | P0 |
| **오답노트 자동 생성** | 틀린 문제 자동 저장 → 재출제 가능 | P0 |
| **취약과목 대시보드** | 과목별 정답률 시각화 | P1 |

### 3.3 스트레치 기능 (2일차, 시간 남으면)

| 기능 | 설명 | 강의 연계 |
|---|---|---|
| **하이브리드 검색** | 법조항(정확 일치)=키워드, 개념 설명(의미)=벡터, 두 방식 결합 | 실습 2~4 (검색 3종) |
| **캐시 적용** | 자주 나오는 질문 빠르게 응답 (Cache-aside + TTL) | 실습 1 (캐시) |
| **답변 근거 검증 (Reflection)** | 근거가 약한 답변은 자기비판 후 재작성 | 실습 4 (서브에이전트) |
| **개념 비교 설명** | 헷갈리는 개념 쌍을 나란히 비교 제시 | 벡터 검색 활용 |

## 4. 기술 아키텍처

```
[Frontend: Next.js + TypeScript]
        │  REST API
        ▼
[Backend: FastAPI]
        │
        ├── 검색 모듈 (키워드 + 벡터 + 하이브리드)
        ├── 캐시 (Redis, Cache-aside·TTL)
        └── ReAct Agent (문서 검색 → 답변 생성 → 근거 검증)
        │
        ▼
[DB: PostgreSQL (pgvector) — 로컬 Docker]
   - 법령/개념 문서 코퍼스
   - 기출문제 은행
   - 사용자 오답 기록
```

### 데이터 흐름 (Q&A 예시)
```
사용자 질문
  → Agent: "문서 DB 먼저 검색" (Thought)
  → 하이브리드 검색 실행 (Act)
  → 결과 근거 부족하면 재검색 또는 답변 보류 (Observe → Reflection)
  → 근거 원문과 함께 답변 반환
```

## 5. 성공 지표 (데모에서 보여줄 수치)

| 지표 | 측정 방법 |
|---|---|
| 캐시 적용 전/후 응답속도 | 실측 ms 비교 (예: 1090ms → 150ms) |
| 검색 방식별 정확도 차이 | 동의어/오타/정확코드 질의 3종에 대한 키워드 vs 벡터 vs 하이브리드 결과 비교표 |
| 오답노트 반영 정확도 | 틀린 문제가 재출제 목록에 정확히 반영되는지 |

> 강의 실습에서 배운 "엔지니어의 눈" 체크리스트(캐시 전/후 실측, 검색 결과 비교)를 그대로 발표 자료에 활용

## 6. 리스크 및 대응

| 리스크 | 대응 방안 |
|---|---|
| 법령 할루시네이션 (틀린 조항 생성) | 반드시 문서 DB 내 원문 근거로만 답변, 근거 없으면 "확인 불가" 표시 |
| 데이터 확보 시간 부족 | 1~2과목만 우선 완성, 나머지는 샘플 데이터로 구조만 시연 |
| 도구 과다호출/무한루프 (Agent) | 호출 횟수 상한 + 종료 조건 명시 (강의 실습 4 참고) |
| 데모 중 예상 밖 질문에 오답 | 발표에서 시연할 질문 3~5개 사전에 검증 완료해둘 것 |
